# Теоретическое задание

## 1. Как обеспечить высокую доступность сервиса

Насколько я понял, если сервис работает только на одном сервере, то он становится точкой отказа — при падении сервера сервис перестаёт быть доступным.
Из того, что я изучил, обычно для повышения доступности используют несколько серверов с одинаковой конфигурацией и ставят перед ними балансировщик нагрузки.
Балансировщик распределяет запросы между серверами, и если один из них перестаёт отвечать, трафик автоматически идёт на остальные.

Примерно это выглядит так:

```text id="h7v7xg"
Пользователь
     ↓
Балансировщик
   ↙   ↓   ↘
server1 server2 server3
```

Также я видел, что:
* используют проверки доступности серверов (health checks)
* настраивают мониторинг, чтобы отслеживать состояние системы
* если есть база данных, её тоже нужно делать отказоустойчивой (например, через репликацию)

Таким образом, основная идея — не полагаться на один сервер, а распределять нагрузку между несколькими.

## 2. Как создать и настроить 5 таких серверов

Для настройки нескольких серверов используют инструменты автоматизации.
Из того, что я увидел, один из самых распространённых вариантов — Ansible.
С его помощью можно один раз описать настройку сервера и применить ее сразу ко всем машинами.

Общий подход я вижу примерно так:
1. Подготовить несколько серверов
2. Настроить доступ по SSH
3. Создать список серверов (inventory)
4. Написать сценарий настройки (playbook)
5. Применить его ко всем серверам

В результате мы настраиваем все сервера одинаково, снижая вероятность ошибиться и ускоряя процесс настройки.
